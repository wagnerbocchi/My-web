Olá, mundo! — por que criei este blog e o que esperar por aqui

autor: Wagner Bocchi
data: 2026-03-03
tags: meta, cybersec, blog
leitura: ~3 min

Todo hello world começa com uma linha de código. Mas este começa com uma confissão: criei esse blog principalmente porque cansei de perder anotações em arquivos .txt espalhados pelo meu Arch.

Quem escreve aqui

Me chamo Wagner Bocchi. Sou um profissional de cibersegurança certificado pela Cisco como Ethical Hacker, atualmente trabalhando como DevOps Engineer. Sou neurodivergente, usuário fanático de Arch Linux e acredito que a melhor forma de aprender é quebrar coisas (de forma ética, claro).

Trabalho na interseção entre segurança ofensiva, DevOps e desenvolvimento em Python. Tenho background em Penetration Testing, SOC/SOAR e um histórico que vai de servidor do Exército Brasileiro a piloto privado — o que me ensinou que disciplina e aprendizado constante são mais valiosos que qualquer ferramenta.

Por que um blog no estilo terminal?

Porque é quem eu sou. Passo a maior parte do meu tempo em terminais, seja fazendo recon num pentest, automatizando um pipeline DevOps ou debugando um script Python às 2 da manhã. O terminal é o ambiente onde me sinto em casa, então nada mais natural que o blog reflita isso.

Tecnicamente: HTML, CSS e JavaScript puro, sem frameworks, sem CMS, sem dependências externas além de uma fonte monospace. Hospedado no GitHub Pages. Simples, rápido, e completamente sob controle.

O que vou escrever

Sem compromisso com frequência, mas com compromisso com honestidade. Tópicos prováveis:

  • Segurança ofensiva — técnicas, ferramentas, writeups de CTFs e vulnerabilidades interessantes
  • Python e automação — scripts úteis, exploits educacionais, automação de processos
  • DevOps e infraestrutura — GCP, Docker, pipelines, N8N e o que mais aparecer
  • OSINT — técnicas de inteligência em fontes abertas
  • Arch Linux e ambiente de trabalho — configurações, dotfiles, ferramentas do dia a dia
  • Outros — sobre ser neurodivergente na área de tech, livros, aviação, cerveja artesanal

Como adicionar novos posts (para mim mesmo)

Copie este arquivo, edite o conteúdo e adicione a entrada correspondente em blog/index.html. Sem banco de dados, sem CMS, sem complicação. 

# Criar novo post
cp blog/posts/primeiro-post.html blog/posts/nome-do-post.html

# Editar conteúdo
nvim blog/posts/nome-do-post.html

# Adicionar entrada no índice do blog
nvim blog/index.html  # adicionar .post-item na lista

# Deploy
git add . && git commit -m "blog: adiciona post nome-do-post"
git push origin main
